# GEMINI.md

## Project Overview

This project is a React + Vite application.

The purpose of this project is to build a modern, responsive company website with a clean and professional design.

The project uses:

- React
- React Router
- Tailwind CSS
- Framer Motion

The architecture is component-based.

Project structure:

src/
    assets/
    images/
    components/
    blocks/
    pages/

---

## General Rules

Before making any changes:

1. Read the relevant files first.
2. Understand the existing architecture.
3. Explain your plan before editing.
4. Make the smallest possible changes.
5. Never rewrite an entire file if only a small modification is needed.

---

## Coding Style

Follow the existing coding style.

Keep:

- Component names
- Folder structure
- File naming convention
- Formatting
- Import order

Do not introduce unnecessary abstraction.

Avoid creating new files unless requested.

---

## UI Rules

Maintain a modern and premium design.

Prefer:

- Responsive layout
- Clean spacing
- Consistent typography
- Smooth animations
- Minimalistic appearance

Do not randomly change:

- Colors
- Font sizes
- Layout spacing

unless explicitly requested.

---

## React Rules

Prefer Functional Components.

Use hooks whenever appropriate.

Avoid unnecessary re-renders.

Keep components modular.

Reuse existing components whenever possible.

---

## Tailwind Rules

Prefer existing Tailwind utility classes.

Avoid inline styles.

Keep spacing consistent.

Use semantic class grouping.

---

## Framer Motion

Reuse the existing animation style.

Do not add excessive animations.

Animations should improve UX.

---

## Images

Do not rename image files.

Do not move assets.

Reuse existing assets whenever possible.

---

## Routing

Preserve the current routing structure.

Do not modify routes unless requested.

---

## Editing Rules

Before editing:

- Explain the problem.
- Explain the solution.
- List affected files.

After editing:

- Summarize all changes.
- Explain why each change was made.

---

## Refactoring

Refactor only when it clearly improves:

- readability
- maintainability
- performance

Never refactor the entire project without permission.

---

## Performance

Avoid unnecessary renders.

Avoid duplicate code.

Prefer reusable components.

Optimize image loading whenever possible.

---

## Git Safety

Never modify unrelated files.

Keep commits focused.

Preserve the existing project architecture.
